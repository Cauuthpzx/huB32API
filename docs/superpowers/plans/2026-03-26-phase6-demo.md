# Phase 6: Demo Mode (Teacher Screen Broadcast) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Teacher clicks "Demo" in the web dashboard → browser captures teacher's screen via `getDisplayMedia()` → produces to mediasoup SFU → all student agents in the room consume the stream and display it fullscreen.

**Architecture:** Teacher browser acts as a mediasoup Producer (using mediasoup-client.js). SFU creates Consumers for each student agent in the room. Agents receive the teacher's stream via libdatachannel and display it in a fullscreen overlay (similar to ScreenLock but with video content).

**Tech Stack:** getDisplayMedia API, mediasoup-client.js, libdatachannel (agent consumer), DirectShow/D3D11 (video rendering)

**Depends on:** Phase 5 (dashboard exists, mediasoup fully working)

### Task 1: Teacher Demo Producer (Browser)

- [ ] **Step 1: Write demo-producer.js**

```javascript
export class DemoProducer {
    constructor(locationId) { this.locationId = locationId; }

    async start() {
        // 1. Capture teacher screen
        this.stream = await navigator.mediaDevices.getDisplayMedia({
            video: { width: 1920, height: 1080, frameRate: 30 }
        });

        // 2. Create mediasoup send transport
        const transportInfo = await api.post('/api/v1/stream/transport', {
            locationId: this.locationId, direction: 'send'
        });
        // ... create producer, similar to agent flow but from browser
    }

    stop() {
        this.stream.getTracks().forEach(t => t.stop());
        // Close transport
    }
}
```

### Task 2: DemoController API

- [ ] **Step 1: Write DemoController**

```
POST /api/v1/demo/start   { locationId } → starts demo mode for room
POST /api/v1/demo/stop    { locationId } → stops demo mode
GET  /api/v1/demo/status  { locationId } → is demo active?
```

When demo starts: server pushes a "demo-start" command to all agents in the room with the teacher's producerId. When demo stops: pushes "demo-stop".
